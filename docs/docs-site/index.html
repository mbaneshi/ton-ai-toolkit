@ton-intent/core

Natural language to TON transactions in 3 lines. Provider-agnostic — swap Claude, GPT, or any custom LLM.

Quick Start

import { IntentParser, TONTxBuilder, createAnthropicProvider } from '@ton-intent/core'

// 3 lines: natural language → TON transaction
const parser = new IntentParser(createAnthropicProvider(process.env.ANTHROPIC_API_KEY))
const intent = await parser.parse("send 5 TON to EQD...abc for design work")
const tx = new TONTxBuilder().buildTransfer(intent)
// tx = { to: Address, value: 5000000000n, body: Cell }

@ton-mcp/server

The only read + write + wallet MCP server for TON. 18 tools. Works with Claude Desktop, Cursor, and any MCP-compatible agent.

Claude Desktop Setup

// ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "ton": {
      "command": "npx",
      "args": ["-y", "@ton-mcp/server"],
      "env": {
        "TONCENTER_API_KEY": "your_key",
        "NETWORK": "testnet"
      }
    }
  }
}

Security Model

Gated Wallet

Wallet tools only available when MNEMONIC env var is set. Mnemonic never exposed in responses.

Amount Limits

Configurable MAX_SEND_TON (default: 1000). Dry-run mode logs instead of broadcasting.

Rate Limiting

Token bucket at 100 req/min on read tools. Retry with exponential backoff on failures.

Natural Language Understanding

No commands needed — just talk naturally in the group:

"send 5 TON to @alice for design work"

Creates a transfer proposal with amount, recipient, and reason auto-extracted.

"add @bob as a signer"

Creates a membership proposal to add a new multisig signer.

"change threshold to 3"

Creates a governance proposal to update the signing threshold.

TON Ecosystem Integration

Deep, production-grade integration with TON blockchain primitives.

Multisig V2

Official ton-blockchain/multisig-contract-v2. Correct TL-B opcodes (0xf718510f). Non-custodial N-of-M signing.

Jetton TEP-74

Full token transfer support with proper opcode (0x0f8a7ea5). Balance queries, metadata reading.

TON DNS

Native .ton domain resolution. Send to domain names instead of raw addresses.

NFT TEP-62

NFT metadata reading and collection querying via MCP tools.

All Address Formats

Raw (0:hex), bounceable (EQ...), non-bounceable (UQ...). Auto-normalization and validation.

TonCenter RPC

Official RPC for testnet and mainnet. Rate limiting, retry with exponential backoff.
